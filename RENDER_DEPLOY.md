# Render Deployment

This project is configured to deploy to Render as a single Node web service that serves both the frontend and the backend API.

## What changed

- The Express server now serves the static frontend from the repository root.
- Frontend API calls now use:
  - `http://localhost:8001/api/v1` during local development.
  - `same-origin/api/v1` when deployed.
- Database initialization can run automatically at startup when `AUTO_INIT_DB=true`.
- `render.yaml` provisions one web service and one PostgreSQL database.

## Deploy with Blueprint

1. Push this repository to GitHub.
2. In Render, choose `New +` > `Blueprint`.
3. Select the repository.
4. Render will read `render.yaml` and create:
   - `virtual-lab` web service
   - `virtual-lab-db` PostgreSQL database
5. Wait for the first deploy to finish.

## Environment notes

- `DATABASE_URL` is injected from the managed Render Postgres instance.
- `JWT_SECRET` is generated by Render.
- `AUTO_INIT_DB=true` runs the schema and seed initializer on startup.
- `DB_SSL=true` and `DB_SSL_REJECT_UNAUTHORIZED=false` are set for Render Postgres.

## Local development

- Frontend can still run separately.
- Backend remains available at `http://localhost:8001`.
- If you serve the HTML locally from a different port, the frontend will still target port `8001` automatically.
